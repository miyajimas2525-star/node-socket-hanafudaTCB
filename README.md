node-socket 花札TCB

ゲームの概要 伝統的な花札の要素を簡略化し、「3枚のカードを選んで役を作る」という戦略性にフォーカスした対戦型ゲームです。プレイヤーが互いのHP（20）を削り合い、先に相手の体力を0にした方が勝利となります。

 使用技術
-フロントエンド: Vanilla JavaScript 
- バックエンド: Node.js, Socket.io
- ロジック: 純粋関数による役判定エンジン

技術的なこだわりと実装のポイント

1. サーバーサイド主導のゲームロジック

クライアントサイドのデータは信用しないという原則に基づき、ゲームの進行、手札の管理、役の判定はすべてNode.jsサーバー側で行っています。

バリデーション: カード提出時、サーバー側で「提出枚数が正しいか」「そのカードを本当に所持しているか」を厳密にチェックし、不正な操作を遮断しています。

整合性の維持: サーバーを「Single Source of Truth（信頼できる唯一の情報源）」とし、各クライアントへ状態をブロードキャストすることで、同期ズレのない対戦を実現しました。

2. 拡張性の高い役判定エンジン

役判定ロジックをクラスとして独立させ、外部状態に依存しない「純粋関数」で構成しました。

判定フロー: 「月下無双」などの特殊役から「カス」などの基本役まで、優先順位に基づいた論理的な判定フローを実装しています。

ボーナス計算: 場の月と一致するカードによる打点加算ロジックなど、花札特有の面白さを計算式に落とし込みました。

3. アルゴリズムと最適化

Fisher–Yatesシャッフル: デッキのシャッフルには偏りのないFisher–Yatesアルゴリズムを採用し、公平なゲーム体験を担保しています。

リソース管理: 捨て札を再利用してデッキを補充するリピートロジックを実装し、長時間の対戦にも耐えうる設計にしました。


実行方法
# リポジトリをクローン
git clone [リポジトリURL]

# 依存関係のインストール
npm install

# サーバー起動
node server.js

その後、ブラウザで http://localhost:3000 を開いてください。
